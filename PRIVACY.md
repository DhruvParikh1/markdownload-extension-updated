# Privacy Policy

This page is used to inform visitors regarding our policies with the collection, use, and disclosure of Personal Information if anyone
using the MarkSnip browser extension (the Service).

tl;dr: MarkSnip keeps clips on-device by default. If you use the optional Notion integration, the clipped markdown and selected metadata are sent to the configured MarkSnip backend and then to Notion only for that save action.

## Information Collection and Use
This extension reads data on a website you are visiting when you click the button. The URL, metadata and content are clipped and fed
through third-party javascript libraries to transofrm it into the Markdown you see in the preview.

For local workflows such as download, copy, batch conversion, and Obsidian export, this data is not sent back to any MarkSnip server.

If you explicitly use the Notion integration:

- the current page's markdown content,
- the page title,
- the source URL,
- the clip timestamp, and
- selected metadata such as tags

are sent to the configured MarkSnip backend so it can securely exchange and refresh Notion OAuth tokens and create the requested Notion page. This data is then sent to Notion to complete the save action.

The backend is intended to treat clip payloads as request-scoped only and not persist them after the Notion request completes.

The extension's general settings are stored in browser storage. Notion connection state is stored locally in the browser and used only to perform user-requested Notion saves.
